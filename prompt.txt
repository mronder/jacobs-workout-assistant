Diagnosis

The Edge Functions bundler is trying to bundle a TypeScript declaration file as an Edge Function. The log shows the edge-functions directory contents lines 65-68 and the specific error: "Default export in '/opt/build/repo/netlify/edge-functions/netlify-types.d.ts' must be a function" line 75. The bundler treats every file in netlify/edge-functions as an edge function; a .d.ts file (type declarations) is not a runnable function, so bundling fails.
The edge functions directory being used is shown in the resolved config line 84.
Relevant log lines: lines 65-68, line 75, line 84.

Cause

A TypeScript declaration file netlify/edge-functions/netlify-types.d.ts (declarations only) was committed inside netlify/edge-functions. Netlify expects files in that folder to default-export a function; a .d.ts file does not, so the build fails.
Solution

Move the declaration file out of the edge-functions directory (recommended).
Example commands:
git mv netlify/edge-functions/netlify-types.d.ts src/types/netlify-types.d.ts
git add -A
git commit -m "Move netlify-types.d.ts out of edge-functions to avoid bundling"
git push
After this change, Netlify will only bundle actual edge function files from netlify/edge-functions and the .d.ts file will no longer be treated as an Edge Function.
Alternative: point Netlify to a different edge functions directory (if you prefer to keep files elsewhere).
Update your netlify.toml to set a different edge_functions path (and ensure only function files are in that folder). Example:
[build]
  edge_functions = "netlify/edge-functions" # change to the folder that contains only functions
Make sure the folder you point to does not contain .d.ts files.
If you intended that file to be a runtime file (not just types), convert it into a proper TypeScript file that exports a function as the default export. Otherwise keep it outside the edge functions folder.
